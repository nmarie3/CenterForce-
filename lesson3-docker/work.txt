Dockerとは
・コンテナ型仮想化技術」であり、アプリケーションを実行するための領域を分割して利用する。
・コンテナは1つのOS上で稼働し、アプリケーションの実行環境を仮想化する。これにより、軽量で高速に起動・停止することができ、リソース消費が少ない。
・コンテナを使用することで、同じハードウェア上で多数のコンテナを効率的に実行可能であり、高い可搬性を持つ。
・メリット：少ないCPUやメモリで動作し、開発と運用の相性が良いためリリースサイクルが高速化する。
・デメリット：複数のホスト間で運用が煩雑になり、カーネルを共有するため個別の変更ができない点。

----------------------------------------------------------
仮想環境の種類と特徴
ホスト型仮想化（VirtualBox、VMware Workstation）
・OS上に仮想化ソフトをインストールし、仮想マシンを稼働させる。
・既存のマシンを利用でき、テスト環境などに便利。
・パフォーマンスはハイパーバイザー型より低く、ホストOSのリソースを消費する。

ハイパーバイザー型仮想化（VMware ESXi、Microsoft Hyper-V）
・ホストOSが不要で、仮想化ソフトウェアがハードウェア上で直接稼働。
・より高いパフォーマンスを発揮し、複数の仮想マシンを効率的に稼働させる。
・専門的な管理知識が必要。

コンテナ型仮想化（Docker）
・1つのOS内で複数のコンテナを稼働させ、リソース消費を最小限に抑えつつ高速に実行できる。
・開発から運用までの流れをスムーズにするが、学習コストが高い。

-------------------------------------------------
コンテナの起動
docker run [オプション] [イメージ名]

コンテナの停止
docker stop [コンテナ名]

コンテナの削除
docker rm [コンテナ名]
